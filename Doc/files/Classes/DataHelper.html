compareString (target, compareTo, ignoreCase)
Indicates whether two strings are identical, optionally ignoring case and safely handling nil values. If either object is not nil and is not a string, returns false.

Parameters

  • target: The first string to compare.
  • compareTo: The second string to compare.
  • ignoreCase: Determines whether case is ignored during comparison.

Usage

  • compareString( nil, nil ) -- true
  • compareString( "Hi", "hi" ) -- false
  • compareString( "Hi", "hi", true ) -- true
  • s = {}; compareString( s, s ) -- false

Return value:

True if both strings are nil or are equal; false otherwise.

memoize (fn, fnIndex)
Returns a memoized version of the specified function, using an optionally specified index function to create a memoization index from the parameters that are passed to invocations of the memoized function. The memoized function will expose a "__forget" method that can be used to clear all memoized results.

Parameters

  • fn: The function to memoize
  • fnIndex: An optional function to create a memoization index from the parameters that are passed to invocations of the memoized function.

Usage

  • inc = memoize( function( a ) print( a ); return a + 1 end )
  • inc( 5 ) -- Executes function, printing "5" and returning "6"
  • inc( 5 ) -- Returns memoized result "6" without executing function
  • inc:__forget() -- Clears all memoized results.
  • inc( 5 ) -- Executes function again, printing "5" and returning "6"
  • sub = memoize( function( a, b ) return a - b end )
  • sub( 3, 1 ) -- Executes function and returns 2
  • sub( 1, 3 ) -- Parameters are in different order, so no memoized result is found. Executes function and returns -2
  • add = memoize( function( a, b ) return a + b end, function( a, b ) return a + b end )
  • add( 1, 3 ) -- Executes function and returns 4
  • add( 3, 1 ) -- Since the specified memoization index function indexes the parameters irrespective of order, returns the memoized result without executing the function again.

Return value:

The memoized version of the specified function.

See also:

  • http://lua-users.org/wiki/FuncTables

selectByNestedIndex (target, ...)
Returns the specified named element of the specified table, traversing nested table elements as necessary. If the specified table is nil, or lacks the specified named element, returns nil.

Parameters

  • target: The target table to search for the specified named element.
  • ...: A list of element names, with each name representing a step deeper into the hierarchy of nested table elements.

Usage

  • selectByNestedIndex( { hi = { bye = 9 } }, "hi", "bye" ) -- 9
  • selectByNestedIndex( { hi = { bye = 9 } }, "hi" ) -- { bye = 9 }
  • selectByNestedIndex( { hi = { bye = 9 } }, "hi", "there" ) -- nil
  • selectByNestedIndex( nil, "hi", "bye" ) -- nil

Return value:

The specified named element of the specified table, if available; otherwise, nil.

  • updateByNestedIndex

updateByNestedIndex (value, target, ...)
Updates the specified named element of the specified table to the specified value and returns it, traversing and creating nested table elements as necessary. If the specified table is nil, returns nil.

Parameters

  • value: The value to which to set the specified named element in the specified table.
  • target: The target table to search for the specified named element.
  • ...: A list of element names, with each name representing a step deeper into the hierarchy of nested table elements.

Usage

  • updateByNestedIndex( 9, { hi = { bye = 0 } }, "hi", "bye" ) -- { bye = 9 }
  • updateByNestedIndex( 9, { hi = {} }, "hi", "bye" ) -- { hi = { bye = 9 } }
  • updateByNestedIndex( { bye = 9}, { hi = {} }, "hi" ) -- { hi = { bye = 9 } }
  • updateByNestedIndex( 9, { hi = {} } ) -- { hi = {} }
  • updateByNestedIndex( nil, "hi", "bye" ) -- nil

Return value:

The specified named element of the specified table, if the table is not nil; otherwise, nil.
