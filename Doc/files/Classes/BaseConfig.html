getDefaultValue (...)
Returns the default value of the specified configuration setting, if it exists.

Parameters

  • ...: A list of configuration setting names, with each name representing a step deeper into the hierarchy of nested configuration settings.

Usage

  • init( { name = "test", tests = { test = 1 } }, { status = "pass" } )
  • getDefaultValue( "name" ) -- nil
  • getDefaultValue( "status" ) -- "pass"

Return value:

The default value of the specified configuration setting, if it exists; otherwise, nil.

  • getValue
  • setValue
  • DataHelper.selectByNestedIndex
getValue (useDefault, ...)
Returns the value of the specified configuration setting, optionally returning a default value if the specified configuration setting is missing.

Parameters

  • useDefault: True if a default value should be returned if the specified configuration setting is missing.
  • ...: A list of configuration setting names, with each name representing a step deeper into the hierarchy of nested configuration settings.

Usage

  • init( { name = "test", tests = { test = 1 } }, { status = "pass" } )
  • getValue( false, "name" ) -- "test"
  • getValue( false, "tests", "test" ) -- 1
  • getValue( false, "status" ) -- nil
  • getValue( true, "status" ) -- "pass"

Return value:

The value of the specified configuration setting, if it exists; otherwise, the default value, if it exists and default values have been specified to be returned.

See also:

  • getDefaultValue
  • setDefaultValue
  • setValue
  • DataHelper.selectByNestedIndex
hasValue (...)
Indicates whether the specified configuration setting exists and has a value other than empty string.

Parameters

  • ...: A list of configuration setting names, with each name representing a step deeper into the hierarchy of nested configuration settings.

Usage

  • init( { name = "test", tests = { test = 1 } }, { status = "pass" } )
  • hasValue( "name" ) -- true
  • hasValue( "tests", "test" ) -- true
  • hasValue( "status" ) -- false

Return value:

True if the specified configuration setting exists and has a value other than empty string; otherwise, false.

See also:

  • getValue
init (values, defaultValues)
Initializes the configuration with the specified values and default values.

Parameters

  • values: The values with which to initialize the configuration.
  • defaultValues: The optional default values with which to initialize the configuration.
setValue (value, ...)
Sets the value of the specified configuration setting.

Parameters

  • value: The value to set for the specified configuratino setting.
  • ...: A list of configuration setting names, with each name representing a step deeper into the hierarchy of nested configuration settings.

Usage

  • init( { name = "test", tests = { test = 1 } } )
  • setValue( "qa", "name" ) -- { name = "qa" }
  • setValue( 2, "tests", "test" ) -- { test = 2 }
  • setValue( 4, "tests", "control", "flag" ) -- { tests = { control = { flag = 4 } } }

Return value:

The new value of the specified configuration setting.
